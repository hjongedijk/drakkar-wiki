# Configuration

Drakkar stores runtime config in:

```txt
/data/config/settings.json
```

## Runtime Layout

Persistent config:

- `/data/config/settings.json` — all integration credentials, API token, and settings

FUSE and media paths (on the host, shared between containers):

- `/mnt/drakkar/vfs` — rclone FUSE mount (WebDAV → NNTP streaming)
- `/mnt/drakkar/media/movies` — Plex movie library (symlinks)
- `/mnt/drakkar/media/tv` — Plex TV library (symlinks)
- `/mnt/.drakkar-vfs-store` — internal VFS backing store (content, completed-symlinks, NZBs)

### Symlink chain

Every media file is a 1-hop symlink from the Plex library into the FUSE mount:

```
/mnt/drakkar/media/movies/Movie (Year)/Movie.mkv
  → /.ids/c/m/p/s/y/<fileId>   (via FUSE)
  → rclone WebDAV GET
  → Drakkar backend
  → NNTP stream
```

## rclone VFS Settings

rclone mounts the backend WebDAV as a FUSE filesystem optimised for streaming:

| Flag | Value | Purpose |
|---|---|---|
| `--vfs-cache-mode` | `full` | Sparse disk cache — only downloaded bytes stored |
| `--vfs-cache-max-size` | `20G` | Disk cache limit |
| `--vfs-cache-max-age` | `24h` | Keep cached files 24 h |
| `--buffer-size` | `128M` | RAM buffer (kept small with full mode) |
| `--vfs-read-ahead` | `1024M` | 1 GB on-disk prefetch ahead of playback position |
| `--vfs-read-chunk-size` | `8M` | 8 MB chunks → first bytes in ~0.6 s |
| `--vfs-fast-fingerprint` | — | Size-only fingerprint, no hash |
| `--no-modtime` | — | Skip modtime reads (files never change) |
| `--no-checksum` | — | Skip checksum verification |
| `--read-only` | — | FUSE is read-only; backend handles writes |
| `--retries` | `0` | Fail fast; NNTP pool handles retries |
| `--low-level-retries` | `0` | Same |
| `--dir-cache-time` | `20s` | Directory cache TTL |

## Services

The public compose includes:

- `drakkar_frontend`, `drakkar_backend`, `drakkar_rclone`
- `drakkar_postgres`, `drakkar_valkey`, `drakkar_valkey_insight`
- `drakkar_nzbhydra2`, `drakkar_apprise_api`, `drakkar_seerr`

All share one bridge network. Integration URLs, API keys, and behaviour are configured via `settings.json`.

## Shared API Token

The Settings page exposes one **Drakkar API Token**.

Use it for:

- frontend to backend requests
- remote REST API calls
- GraphQL requests

```bash
curl -H 'x-api-token: YOUR_DRAKKAR_API_TOKEN' \
  http://HOST:8080/api/status
```

`Authorization: Bearer YOUR_DRAKKAR_API_TOKEN` also works.

## Monitoring Tasks

All tasks run unconditionally on schedule (no deferral for streaming or IO):

| Task | Default interval |
|---|---|
| `request-download-recovery` | 10 min |
| `nzbhydra-rss-sync` | 15 min |
| `background-repair` | 30 min |
| `log-prune` | 6 h |

`monitorQueueSeedTarget` (default 20) controls how many missing items are queued per cycle.

## Request Sync Cadence

Two paths handle request intake:

- **Seerr webhook** — immediate delivery for single approved requests
- **Periodic request sync** — broader reconciliation, duplicate cleanup, and recovery every 10 minutes

## NZBHydra Categories

Default categories:

- Movies: `2030`, `2040`, `2045`, `2050`, `2060`
- TV: `5030`, `5040`, `5045`, `5080`
