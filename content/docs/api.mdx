# API

## Interactive Docs (Scalar)

Drakkar ships an interactive API reference at:

```txt
http://HOST:3000/api/docs
```

All endpoints, request/response schemas, and authentication requirements are documented there.

## Authentication

All API calls require the **Drakkar API Token** found in Settings.

```bash
curl -H 'x-api-token: YOUR_DRAKKAR_API_TOKEN' \
  http://HOST:8080/api/status
```

`Authorization: Bearer YOUR_DRAKKAR_API_TOKEN` also works.

## Key REST Endpoints

| Endpoint | Description |
|---|---|
| `GET /api/status` | System health, queue sizes, active streams |
| `GET /api/library` | Media library items |
| `GET /api/requests` | Request list |
| `GET /api/downloads/queue/page` | Download queue |
| `GET /api/tasks` | Scheduled task status and next run times |
| `GET /api/vfs/streams/metrics` | NNTP streaming metrics and cache hit rate |
| `POST /api/webhooks/seerr` | Seerr webhook for immediate request intake |
| `/api/compat/sonarr` | Sonarr-compatible endpoint (for Bazarr etc.) |
| `/api/compat/radarr` | Radarr-compatible endpoint |

## GraphQL

Interactive GraphiQL at:

```txt
http://HOST:8080/api/graphql
```

Example query:

```graphql
{
  status { version backend postgresql valkey }
  downloads(limit: 5) { title status progress }
  library(limit: 5) { title mediaType libraryStatus updatedAt }
}
```

## Seerr Webhook

Configure in Seerr to point at:

```txt
http://YOUR_HOST:8080/api/webhooks/seerr
```

Enables immediate ingest of single approved requests without waiting for the periodic sync cycle.

## Arr Compatibility (Bazarr)

Drakkar exposes Radarr/Sonarr-compatible endpoints for Bazarr:

- `/api/compat/sonarr`
- `/api/compat/radarr`

Use the Drakkar API token as the Bazarr API key. Both `apikey=` query param and `X-Api-Key` header are accepted.
